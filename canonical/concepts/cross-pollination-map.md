# Cross-Pollination + Shared IP — Adrian's 5-Business Stack

## 0. Why this exists

Five businesses look like five businesses to an outside observer. They are not. They are one **founder operating system** that uses the same physical substrate, the same methodologies, the same network of people, and the same voice across every offering. The cross-pollination is the moat (P12 — concentrated bets, durable moats); it is also the failure mode (P4 — decisive product focus), because what is shared can also be confused.

This document maps the shared layer rigorously so cross-pollination decisions are deliberate, not accidental.

## 7. Application via P1–P15 lens

**P1 (founder = conductor)**: every combo offering must run on lateral handoffs, not on Adrian-as-executor. Test each combo against "does this require Adrian to do anything operational beyond direction?" Reject anything that fails.

**P3 + P10 (founder control / not co-founded)**: Combo 3 (AYA × SS) requires explicit IP licensing to AYA, with Adrian retaining rights to use SS methodology elsewhere. Do not let SS methodology become AYA-owned.

**P4 (decisive product focus)**: Combo 5 (Methodology Stack) is the most ambitious; P4 says don't ship it until all five projects are individually robust. **Sequence: each business reaches its own MVP before any combo is sold.**

**P5 (insist on quality)**: every combo carries the worst component's quality through the whole bundle. Audit each combo's weakest link before pricing.

**P12 (concentrated bets, durable moats)**: cross-pollination IS the moat. Continue investing in shared substrate (Siberian Blue supply, AGA land, methodology IP) — these compound. Resist diversification into unrelated new businesses.

**P13 (work backwards from customer)**: write the customer-testimonial-2-years-out for each combo before building it. If the testimonial isn't vivid, the combo isn't real.

**P14 (direction over efficiency)**: every cross-pollination decision asks: is this serving founder mission, or is it convenient internal logic?
