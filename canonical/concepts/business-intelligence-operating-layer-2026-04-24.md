## What this index is

On 2026-04-24, the vault gained a **Business Intelligence (BI) layer** — one execution-grade canonical doc per active business. Each BI doc follows a 10-section format: executive summary, current state + blockers, voice anchor, ICP, IP asset map, top 5 historical decisions, next 3 strategic actions, marketing playbook, content playbook, testimonial library.

This index is the navigation table for that layer. It tells you which BI docs exist, their current depth, what execution packs sit downstream of each, and what gaps remain.

**For ecosystem-level CEO synthesis**, read `canonical/ecosystem/overview.md` first. **For execution within one business**, read that business's BI doc + its execution pack.

## How a fresh agent uses this index

1. Read this file.
2. Identify which business(es) the user's question touches.
3. Read the relevant BI doc(s) — the 10-section structure is consistent across all of them.
4. If the question is execution-oriented (what to publish, what to send, what to draft), read the relevant execution pack.
5. If the question is cross-business or strategic, read `canonical/ecosystem/overview.md` for the CEO-altitude framing.
6. If the BI doc is silent on the question and the wider canonical is silent too, **flag the gap rather than fabricating the answer**.

## How Adrian uses this index

1. **For weekly execution:** open this index → click into the relevant execution pack → publish.
2. **For investor / strategic conversations:** open this index → cross-reference cross-business themes (above) → use the BI docs as briefing material.
3. **For onboarding a new agent or assistant:** point them at this file plus `canonical/INDEX.md` plus the relevant project's BI doc. That's enough for them to be useful within an hour.
