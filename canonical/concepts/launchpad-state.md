# Launchpad State

## Purpose
The **Launchpad State** block is a ledger-native state machine for tracking each venture's launch readiness. It is explicitly **not** a marketing plan or a static business plan. It forces the venture to continually answer five disciplined questions (Who is it for? What problem does it solve? How? Which launch quadrant is next? What evidence is missing?) and surfaces operational gaps.

## When to Populate / Re-Run
- **Idea Stage:** When a new venture is conceived.
- **Stage Transitions:** At each major transition (e.g., concept → pre-revenue → revenue-active → scale).
- **State Changes:** When a launch quadrant shift occurs or an evidence gate state materially changes.
- **Review Cadence:** Minimum quarterly review for all active ventures.

## What It Deliberately Is NOT
- **Not a rigid niche-statement template.** The standard freelancer formula ("I help X solve Y through Z") breaks for luxury products (OSB), place-based communities (AGA), B2B infrastructure (Tri Hita), and pre-category research platforms (Ashta).
- **Not a freelancer acquisition plan.** It does not optimize for hourly billing, mass cold-email, or Upwork proposals.
- **Not a substitute for the project ledger.** It is a *section* within the ledger, complementing the broader strategic truth.
- **Not external canonical truth.** It is an internal state tracking mechanism.

## Anti-Pattern Warning
Do not apply generic agency tactics (e.g., Upwork scraping, cold-email blasts, niche-formula rigidity) to luxury, spiritual, multi-stakeholder, or B2B-regulatory ventures. The Launchpad State is a discipline, not a template to fill literally. It accepts that some answers are "not yet" and focuses on identifying what needs to be validated next.
